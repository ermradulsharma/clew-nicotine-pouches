# Contributing to Clew Pouches

Thank you for your interest in contributing to the Clew Pouches project! We welcome contributions that help improve the platform.

## Getting Started

1. Fork the repository.
2. Clone your fork locally: `git clone <your-fork-url>`
3. Set up the environment following the instructions in the [README.md](README.md).

## Development Workflow

### Branching Strategy

- `main`: Production-ready code.
- `develop`: Ongoing development.
- Feature branches: `feature/your-feature-name`
- Bug fixes: `fix/your-bug-name`

### Coding Standards

- Follow [PSR-12](https://www.php-fig.org/psr/psr-12/) coding standards for PHP.
- Ensure all new features are accompanied by appropriate tests (if applicable).
- Keep code clean, readable, and well-documented.

### Pull Request Process

1. Create a new branch for your changes.
2. Ensure your code passes all tests.
3. Commit your changes with descriptive commit messages.
4. Push your branch to your fork.
5. Open a Pull Request against the `develop` branch of the main repository.
6. Provide a clear description of the changes and link any related issues.

## Reporting Issues

Use GitHub issues to report bugs or suggest new features. Please use the provided templates (if available) and provide as much detail as possible.
